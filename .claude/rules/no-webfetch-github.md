# github.com／raw.githubusercontent.comに対するWebFetchの禁止

このルールはsettings.local.jsonによって強制的に適用される。

WebFetchは要約を返すツールであり一次情報を直接得られないため、ソースコードやドキュメント原本を取得するタスクには不適切である。GitHubにアクセスして取得したいリソースはほとんどの場合ソースコードやドキュメント原本であり、`git clone`あるいは`gh`（いずれも利用不可の場合はAPI）によって取得可能であるため、WebFetchを一律で禁止する。

## 禁止された際の回避行動の禁止

このルールでWebFetchが拒否された場合に、GitHub以外のソース（ミラー、第三者による再配布、要約記事など）から代替情報を取得しようとする回避行動を禁止する。

これらの回避行動は単なる手段の制約違反のみならず、タスクの目的そのものを毀損する行為である。タスクの本来の目的に立ち返れ。本来必要なのは当該リポジトリの一次情報（正規かつ最新のソースコード・ドキュメント原本）であって、リポジトリの直接取得以外による情報取得は代替にはなりえない。
